# UI Example

A simple UI design is included for the 'dummy' state which is accessible on the train\_crud runner, including a text editor, a select editor, a Markdown editor and a Json editor for the full data model.

<figure><img src="../../.gitbook/assets/image (44).png" alt=""><figcaption><p>Tabs Example</p></figcaption></figure>

The UI is configured to use a table lister, where the list of columns and the filter parameters which can be used are also defined.

<figure><img src="../../.gitbook/assets/image (45).png" alt=""><figcaption><p>Lister Example</p></figcaption></figure>

On the Source screen, train\_crud/dummy endpoint is defined as the data source for all actions, meaning standard CRUD methods will be used automatically to feed this UI.&#x20;

<figure><img src="../../.gitbook/assets/image (46).png" alt=""><figcaption><p>Source Example</p></figcaption></figure>

A 'training' App is also defined, with only the 'dummy' UI included in its menu.

<figure><img src="../../.gitbook/assets/image (48).png" alt=""><figcaption><p>Training App Design</p></figcaption></figure>

Navigating to \[UI\_SERVER]/app/train, you can see this single UI app in action.

<figure><img src="../../.gitbook/assets/image (50).png" alt=""><figcaption><p>Training App</p></figcaption></figure>
